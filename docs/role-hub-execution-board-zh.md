## 4. 日会节奏与沟通协议

### 4.1 日会制度

| 项目 | 规格 |
|---|---|
| **频率** | 每工作日 1 次，15 分钟 |
| **时间** | 建议 10:00 AM（团队统一时区） |
| **形式** | 同步站会（视频/文字均可） |
| **参与者** | 全部 6 名 worker |
| **主持** | worker-1（PM/交付负责人）轮值 |

### 4.2 日会议程模板

```
1. 每人 2 分钟更新（≤12 分钟）
   - 昨日完成
   - 今日计划
   - 阻塞/风险（有则说明，无则跳过）

2. 阻塞处理（≤3 分钟）
   - 识别阻塞 → 指定 owner → 约定解除时间
   - 阻塞超过 24h 未解决 → 升级至主控

3. 看板更新
   - worker-1 在会后 10 分钟内更新本文档任务状态
```

### 4.3 升级协议

| 级别 | 条件 | 动作 | 响应时效 |
|---|---|---|---|
| **L0 日常** | 任务进展正常 | 日会同步即可 | 次日 |
| **L1 阻塞** | 单任务阻塞 > 24h | worker-1 介入协调 | 当日 |
| **L2 风险** | 关键路径任务延迟 > 2 天 | worker-1 向主控报告 + 制定恢复方案 | 4 小时内 |
| **L3 危机** | 里程碑整体延迟或 P0 质量事故 | 主控决策会 | 2 小时内 |

### 5.2 检查点审查流程

```
1. 检查点 owner 提交审查材料（测试报告/指标截图/演示视频）
2. 参与方异步 review（≤4h）
3. 同步审查会（≤30min）

4. worker-1 记录结论并更新本文档
5. 不通过项指定 owner + 修复期限 → 复检
```

## 10. 附录：状态定义

### 10.1 任务状态

| 状态 | 含义 |
|---|---|
| `pending` | 未开始，等待前置条件 |
| `blocked_by:X` | 被任务 X 阻塞 |
| `in_progress` | 正在执行 |
| `in_review` | 待评审 |
| `done` | 完成并通过验收 |

### 10.2 风险状态

| 状态 | 含义 |
|---|---|
| `identified` | 已识别，尚未分析 |
| `monitoring` | 已有缓释方案，持续监控 |
| `active` | 已触发或高概率触发，需立即行动 |
| `mitigated` | 已缓释，降至可接受水平 |
| `closed` | 已关闭 |
