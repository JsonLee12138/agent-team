## Emergency Response Plan

**If negative feedback dominates:**
- Acknowledge valid criticism immediately
- Don't get defensive — thank people for the feedback
- Create GitHub issues for actionable suggestions
- Post a follow-up: "We heard you. Here's what we're fixing."

**If a competing tool launches same week:**
- Don't mention competitors directly
- Emphasize unique differentiators (Git worktree isolation, brainstorming gates)
- Focus on community engagement over feature comparison

**If content goes unexpectedly viral:**
- Ensure GitHub repo is ready (README, LICENSE, CONTRIBUTING.md)
- Prepare for issue/PR volume surge
- Have a "Thank you" post drafted and ready
