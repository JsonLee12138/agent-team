# Task: role-repo find 入库规则校正

## 背景
- 当前 `role-repo find` 在引入写入数据库逻辑时，需要明确可写入数据来源。

## 目标
- 仅当查询结果来自 GitHub 且包含完整路径信息时，才允许写入数据库。
- 本地角色（local roles）不允许写入该数据库链路。

## 规则定义
1. 数据来源必须是 GitHub 查询结果。
2. 记录必须包含可用的完整路径（用于唯一定位 role）。
3. 不满足上述条件的记录一律跳过，不入库。
4. 本地角色发现结果一律不入库。

## 验收标准（后续开发时使用）
- `find` 结果中，只有符合“GitHub + 完整路径”条件的数据被写入数据库。
- 本地角色数据在任何情况下都不会进入数据库。
- 对被跳过数据有明确日志或统计（可观测）。
